# Onboarding observation sessions

No telemetry in V1. Validate onboarding with 3–5 shoulder sessions using target-persona users.

## Round one (after Welcome → example → gate ship)

Measure time to understand the product promise and complete agent setup (or IT handoff).

- Can the user explain what tamtri does after the pre-recorded example?
- Does the single recommended gate card beat browsing the full roster?
- Does IT checklist copy get sent?

Reorder steps 6–8 in the implementation plan from findings.

## Round two (after sample + own-file flow)

Targets:

| Milestone | Target |
|-----------|--------|
| Sample worked | First rendered artifact from bundled CSV within one session |
| My file worked | Same user runs their own file and sees artifact auto-open |

Record: time to first artifact, permission-card confusion, Finder reveal discovery.

## How to run a session

1. Fresh vault (or new Mac user account).
2. Packaged desktop build or `pnpm run dev:desktop`.
3. Do not coach unless stuck >2 minutes.
4. Note verbatim quotes for copy fixes.
